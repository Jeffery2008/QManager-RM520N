# 汉化版版本号策略

本仓库的汉化版本不再通过改动上游的大版本号来区分。

版本规则：

- 上游原版标签：`vX.Y.Z`
- 本仓库汉化标签：`vX.Y.Z-cn.N`

示例：

- 上游原版：`v0.1.5`
- 本仓库首个汉化发布：`v0.1.5-cn.1`
- 同一上游版本下的下一次汉化修订：`v0.1.5-cn.2`
- 当上游升级到 `v0.1.6` 后，本仓库对应的首个汉化发布：`v0.1.6-cn.1`

设计目标：

- 与原仓库大版本号保持同步
- 明确区分“同一上游版本下的汉化修订”
- 让更新程序能识别：
  - `v0.1.5-cn.2` 新于 `v0.1.5-cn.1`
  - `v0.1.5-cn.1` 新于 `v0.1.5`
  - `v0.1.6-cn.1` 新于 `v0.1.5-cn.9`

实现说明：

- `scripts/www/cgi-bin/quecmanager/system/update.sh` 中的版本比较逻辑已扩展，支持 `-cn.N` 作为汉化修订号比较。
- OTA、检查更新、版本选择等前端界面直接显示完整标签。
- 安装脚本中的内部版本号也应与 release tag 保持一致。
